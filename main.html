Developer Reference

File Guide

An overview of every file in this project — what each page does, which stylesheet controls it, and how the JavaScript is wired together.

🎨 CSS Stylesheet Groups

Each stylesheet governs a specific set of pages. All pages also pull from main.css for base resets, the navbar, buttons, footer, and global utilities.

main.css
Controls the entire landing page (main.html): CSS variables, resets, navbar, hero section, Learn More cards, Options tiles, Help cards, footer, scroll reveal animations, and responsive breakpoints. Also acts as the shared base imported by every sub-page.
main.html + all sub-pages (base)
options.css
Controls the shared layout for all Options sub-pages: the page hero banner, back link, placeholder cards, shimmer loading states, filter bar, and the full schedule timetable (used by schoolsched.html).
enrollment.html academics.html map.html calendar.html faculty.html clubsnorgs.html forms.html announcements.html schoolsched.html
help.css
Controls the shared layout for Help sub-pages: the page hero, FAQ accordion items and open/close animation, contact info cards, the map embed placeholder wrapper, and the placeholder banner used before content is live.
faq.html contactinfo.html
wip.css
Controls the full-screen WIP overlay that locks every sub-page behind a password prompt. Includes the backdrop, animated box, input field, shake keyframe, and fade-out exit animation. Toggled by main.js → initWIP().
All sub-pages (overlay)

⚡ main.js — All JavaScript

A single script handles all interactive behavior across the entire site. It is loaded by every page via <script src="main.js"></script> at the bottom of each file.

Navbar scroll Adds the .scrolled class to #navbar after scrolling 60px, triggering the frosted-glass background. Also highlights the active nav link based on scroll position.
Hamburger menu Toggles the .open class on #hamburger and #mobileMenu. Mobile links close the menu on click.
Scroll reveal Uses IntersectionObserver to add the .visible class to .card, .option-tile, .help-card, and .section-header elements as they enter the viewport, triggering their fade-up animation from main.css.
Smooth scroll Intercepts all href="#..." anchor clicks and scrolls smoothly with a 70px navbar offset.
initWIP() — WIP Overlay Runs on DOMContentLoaded. If a page has #wipOverlay, the overlay is shown and scroll is locked. On correct password (oracle), sets sessionStorage('wip_bypass', '1') and fades the overlay out. Subsequent navigations skip the prompt for the entire session. Wrong passwords trigger a shake animation on #wipBox.
